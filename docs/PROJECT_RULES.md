# WebDiag project rules

1. No fact may be invented to fill a gap. Unknown and unverified information must be marked internally as such.
2. Code is not described as working until it has been executed by a relevant test or build command.
3. A public release is blocked until all 110 declared tools and every declared product feature pass their release gates.
4. Internal implementation states must not appear in public navigation, public tool pages, sitemaps, or user-facing copy.
5. Public copy must use clear language and must describe only observable product behaviour.
6. No fabricated customers, reviews, cases, metrics, accuracy claims, test results, competitors, domain status, or technical capabilities.
7. Russian and English content require independent editorial review.
