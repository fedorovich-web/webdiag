# DESIGN.md — initial constraints

Статус: direction constraints, не финальная visual concept.

- Профессиональная техническая платформа, а не generic AI SaaS.
- Никаких purple-blue AI gradient, glowing orb и бессистемного bento.
- Каталог из 110 инструментов обязан оставаться навигационно понятным.
- Нужны search, categories, recently used/favorites только после фактической реализации.
- Один design system и одна icon system.
- Design tokens — source of truth.
- WCAG 2.2 AA.
- Tool pages используют общий shell, но не должны выглядеть как 110 полностью одинаковых thin pages.
- Core product получает более высокий visual priority, чем каталог утилит.

## Bilingual and editorial layouts

- Components tolerate Russian and English text expansion.
- No fixed-width CTA labels.
- Font system fully supports Cyrillic and Latin.
- Accessible language switcher is mandatory.
- Blog layouts support long-form typography, code blocks, tables, figures and source notes.
